เพจต้นแบบ BBC News

Header นี้น่าจะเริ่มด้วย block ที่ไม่มี margin ซ้าย-ขวา(ขนาดเท่า viewport) แล้วให้ element ที่เป็นลูกทันทีแสดงผลแบบ block และมี max-width ขนาดนึง พร้อมกับให้มี margin ซ้าย-ขวา auto เพื่อให้ block ลูกเหล่านั้นอยู่ตรงกลางจอ

ส่วนของ navigation bar คิดว่าใช้การแสดงผลแบบ flex โดยมี flex flow แบบ row และ nowrap เพราะสมาชิกจะอยู่ติดกันบนบรรทัดเดียวกันตลอด แต่ของเขามีการตอบสนองกับขนาดของ viewport เป็นพิเศษด้วยการหยุดการแสดงผลของสมาชิกใน flex บางตัวเมื่อขนาดหน้าจอเล็กเกินไป แล้วยังมีการเปลี่ยน navigation bar ให้กลายเป็นเมนู dropdown ด้วยเมื่อ viewport เล็กมากๆ ซึ่งในตัวอย่างที่จะแสดงต่อไปขอละไว้

ในส่วนของกลุ่มพาดหัวข่าวแรก เห็นครั้งแรกคิดว่าใช้ grid ในการจัดวาง แต่พอลองปรับขนาด viewport ดูแล้วเห็นว่า เพจมีการตอบสนองต่อการเปลี่ยนขนาด viewport ข้อนค่างซับซ้อน ซึ่งจริงๆก็ไม่มั่นใจว่าทำได้อย่างไร แต่ในที่นี้คิดว่าใช้ flex น่าจะดีกว่า grid
โดยจะใช้ flex flow แบบ row และ wrap ในส่วนของสมาชิกจะกำหนดให้สมาชิกธรรมดา(แบบที่มีรูปข้างบนแล้วมีข้อความด้านล่าง)มีความกว้างขนาดนึง เทียบเป็นความกว้าง 1 บล๊อค
โดยใน div flex จะมีความกว้างพอที่จะรองรับได้ 5 บล๊อคเป็นอย่างมาก และนอกจากสมาชิกธรรมดาที่กล่าวไปแล้ว กลุ่มพาดหัวข่าวนี้จะมีสมาชิกพิเศษอีกสามแบบ

แบบที่หนึ่งคือสมาชิกแรกสุด คิดว่าการจัดวางที่ควรทำคือ เริ่มจากจับกลุ่มรูปภาพและกลุ่มข้อความให้อยู่ div เดียวกัน แล้วให้ div นี้แสดงผลเป็น flex โดยมี flow แบบ row-reverse และ wrap เพื่อให้รูปภาพที่มาก่อนในเอกสารไปอยู่ทางขวาของข้อความ โดยรูปภาพของสมาชิกนี้จะมีความกว้างอยู่ที่ 2 บล๊อค และกลุ่มข้อความจะมีความกว้าง 1 บล๊อค ทำให้สมาชิกนี้มีความกว้างโดยรวม 3 บล๊อค นอกจากนี้สมาชิกนี้ยังต้องมี div ที่มีข้อความสามข้อความข้างล่างด้วย โดยเพื่อให้ทั้งสามข้อความอยู่ในบรรทัดเดียวกัน div นี้จึงต้องเป็น flex ด้วย โดย flow แบบ row และ wrap

แบบที่สองคือสมาชิกที่มีรูปภาพยาวกว่าสมาชิกทั่วไปและข้อความมีแต่พาดหัวข่าว คิดว่าวิธีนึงที่จะทำแบบนี้ได้คือให้สมาชิกนี้มีโครงสร้างใน html เหมือนกับสมาชิกธรรมดา แต่ให้เนื้อหาย่อของข่าวกับเวลาลงข่าวไม่แสดงผล ในส่วนของ div ของรูปภาพแน่นอนว่าต้องให้มี aspect ratio มาทางแนวตั้งมากกว่าแนวนอน และในส่วนของ div ของกลุ่มข้อความก็ให้มี position: absolute; แล้วเลื่อนตำแหน่งลงมาจากขอบบนซัก 70% ก็น่าจะพอได้

แบบที่สามคือสมาชิกที่มีหลายพาดหัวข่าวแต่ไม่มีรูปภาพ คิดว่าวิธีนึงที่ทำได้คือนำสมาชิกธรรมดา 4 ตัวมารวมไว้ใน div เดียวกัน โดยให้ div นี้แสดงเป็น flex และมี flow แบบ row และ wrap โดยมีความกว้าง 1 บล๊อคหรือมากกว่านั้น(ให้มี flex-grow) ในส่วนของสมาชิกธรรมดาที่นำมารวมไว้ก็ให้ซ่อนการแสดงผลของ รูปภาพและเรื่องย่อของข่าวไว้ และให้ตัวมันมีความกว้างได้แค่ 1 บล๊อค
ที่สมาชิกนี้ต้องทำเป็น flex ไว้ก็เผื่อในกรณีที่สมาชิกนี้ตกไปอยู่บรรทัดถัดไปที่มีที่ว่างเหลือทางขวา สมาชิกธรรมดาทั้ง 4 ตัวจะได้ขึ้นมาชิดติดกันตามแนวนอนได้

ในส่วนของ footer นี้น่าจะทำงานคล้ายกับ header โดยเริ่มด้วย block ที่ไม่มี margin ซ้าย-ขวา(ขนาดเท่า viewport) แล้วให้ element ที่เป็นลูกทันทีแสดงผลแบบ block และมี max-width ขนาดนึง พร้อมกับให้มี margin ซ้าย-ขวา auto เพื่อให้ block ลูกเหล่านั้นอยู่ตรงกลางจอ
'Explore the BBC' น่าจะใช้ h2 ธรรมดา

ในส่วนของ navigation bar (ที่มี Home, News, Sport, ...) ของต้นแบบเขาก็มีการตอบสนองกับขนาด viewport ที่ซับซ้อนอีกแล้ว แต่ถ้าจะเอาให้พอเหมือนคิดว่าน่าใช้ flex ที่มี flow แบบ row และ wrap เนื่องจากมีการขึ้นบรรทัดใหม่ และใช้ justify-content: space-between; เพื่อให้แต่ละสมาชิกกระจายตัวไปชิดฝั่งซ้ายขวา และมีความห่างเท่าๆกัน โดยจะต้องตั้ง flex-basis ที่เท่ากันให้ทุกๆสมาชิก สมาชิกแต่ละตัวจะได้มีความกว้างเท่ากัน โดยไม่สนว่าข้อความของแต่ละสมาชิกจะยาวขนาดไหน

สำหรับบรรทัดถัดไปที่รวมรายการลิ้งค์(ที่มี Terms of Uses, About the BBC, ... คล้ายๆ navigation bar อีกแล้ว) น่าจะใช้ flex ที่มี flow แบบ row และ wrap ได้เลย แล้วใส่ column-gap เพื่อแยกแต่ละลิ้งค์ออกจากกัน
